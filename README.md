# Анализ эффективности маркетинга и вовлеченности (Mobile Game Case)

Проект посвящен комплексному анализу метрик мобильной игры. В центре исследования — оценка эффективности мартовской маркетинговой акции, анализ виральности продукта и сегментация лояльной аудитории.

## 🎯 Задачи проекта
* **Очистка данных и DQ-аудит:** локализация технических багов логирования сессий на платформе iOS.
* **Анализ вовлеченности:** расчет Retention-метрик (DAU, WAU, MAU) и Sticky Factor.
* **Финансовый анализ:** оценка эластичности спроса при изменении цен и расчет ARPU по когортам.
* **Проверка гипотез:** сравнение лояльности различных каналов привлечения трафика (A/B тест стратегий закупки).
* **Виральность:** расчет K-factor и прогнозирование органического роста клиентской базы.

## 🛠 Стек технологий
* **Python (Pandas, Plotly)** — обработка данных и интерактивная визуализация.
* **DuckDB** — высокопроизводительный движок для аналитических SQL-запросов (In-memory OLAP).
* **SQL (PostgreSQL dialect)** — сложные JOIN-ы с историческими ценами и оконные функции.
* **Google Colab** — среда для воспроизводимого исследования.

## 📁 Структура репозитория
* `data/` — исходные датасеты в формате CSV (транзакции, логи сессий, цены, пользователи).
* `notebooks/mobile-game-performance-analysis.ipynb` — основной процесс исследования (EDA, расчет метрик, выводы).
* `README.md` — описание проекта и ключевых результатов.

## 📊 Краткий вывод Анализа:
Внедрение мартовской акции с ежедневными наградами привело к росту **Sticky Factor с 15% до 25%** и рекордному увеличению выручки, несмотря на январское повышение цен. Выявлен критический баг в логировании сессий на iOS (11.7% ошибок), требующий технической правки для корректного расчета LTV.
